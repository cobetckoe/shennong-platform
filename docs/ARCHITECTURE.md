# 神农平台开源架构

## 项目概述

神农平台是一个城市农业的分布式AI生产交易平台，目标是让成百台智能种植设备形成一个可扩展、可迭代、可运营的生态系统。

系统由以下部分组成：

- 设备端：智能种植机、网关、传感器、执行单元
- 通信层：Meshtastic Mesh 网络、Tuya 生态、云端连接
- 本地智能层：T5 边缘网关、duckyclaw、实时调参模型、TF 卡数据存储
- 云端智能层：数据备份、模型训练、分析、APP 控制、商品交易
- 用户交互层：移动 APP、云端仪表盘、本地屏幕/按键控制

## 关键模块

### 1. 设备端模块

#### 1.1 种子自动播种机

- 功能：实现精确播种与定位、播种量控制、土壤湿度和环境检测、自动灌溉输出
- 传感器：土壤湿度、温度、光照、风速、CO2、EC
- 执行器：步进电机、播种机构、电磁阀、雾化喷头
- 智能点：根据作物类型自动调整播种间距与灌溉策略

#### 1.2 蔬菜气雾种植机

- 功能：气雾栽培、营养液喷淋、环境控制、光周期管理
- 传感器：温度、湿度、光照、营养液 EC、pH
- 执行器：喷雾泵、风扇、LED 生长灯、营养液循环泵
- 智能点：基于品种迭代喷雾频率与营养配比

#### 1.3 土豆种植机

- 功能：适配块茎作物的栽培、根系支持、分阶段控水控养
- 传感器：根区湿度、温度、溶解氧、营养浓度
- 执行器：雾化喷头、加压泵、支撑机构
- 智能点：土豆生长阶段自动调节喷雾强度与根系通气

#### 1.4 螺旋藻种植机

- 功能：微藻培养环境控制、光照与搅拌、营养与CO2控制
- 传感器：温度、光照、pH、溶氧、浊度
- 执行器：LED 生长灯、搅拌电机、气泵、投料机构
- 智能点：根据培养阶段自动调整光照强度和搅拌模式

#### 1.5 蘑菇种植机

- 功能：蘑菇基质培养控制、喷雾与温湿度调节、CO2 管理
- 传感器：温度、湿度、CO2、光照、基质湿度
- 执行器：喷雾系统、风扇、加热板、补水泵
- 智能点：根据菌蕾阶段自动调整环境与水分

### 2. 固件层模块

固件采用**平行双环境架构**：T5 网关（TuyaOS）和 Arduino 子设备（PlatformIO）是两个独立的开发环境，各自编译部署。

#### 2.1 T5 AI边缘种植网关（独立环境1）

- **平台**: 涂鸦 T5 + TuyaOS
- **编译**: CMake + make
- **功能**:
  - 基于涂鸦 T5 平台，实现边缘计算与业务逻辑载入
  - 内置 `duckyclaw`，负责本地控制策略与智能调参
  - 承载 Meshtastic 通信客户端，维护设备间网格数据流
  - 提供本地按键与显示器接口，支持脱网运行

#### 2.2 Arduino 子设备（独立环境2）

- **平台**: Arduino / STM32 等微控制器
- **编译**: PlatformIO
- **功能**:
  - 按生物分类组织的多种设备（种子播种机、蔬菜气雾、土豆、螺旋藻、蘑菇）
  - 各设备独立采集传感器数据、控制执行器
  - 通过 Meshtastic 模块与 T5 网关通信

#### 2.3 Meshtastic 数据网络

- 构建设备间低功耗长距离网格通信
- 负责收集各个种植机传感器数据、状态心跳、控制指令
- 支持设备数量级扩展，适配数百台种植机

#### 2.4 本地数据存储与备份

- TF卡存储：传感器数据、设备日志、模型参数、运行历史
- 云端同步：定时或事件式上传原始数据与训练输出来备份
- 本地缓存：网络不可用时继续采集与控制

#### 2.5 云端与本地协同控制

- 本地按键：在现场直接切换模式、启动/停止、手动控制
- 本地显示器：实时显示环境数据、运行状态与警报
- 远程APP：作物选择、设备状态、手动/自动控制、交易入口
- 云端：统一管理设备配置、推送参数、分析结果

### 3. AI模型模块

#### 3.1 本地调参模型

- 输入：实时传感器数据、作物类型、生长阶段、历史曲线
- 输出：喷雾周期、光照强度、温湿度控制、营养液配比
- 算法：小型神经网络 / 强化学习 /规则增强模型，支持在线迭代优化
- 目标：每台设备独立自学习，降低人工干预，实现自适应控制

#### 3.2 云端训练与分析

- 数据汇总：云端整合来自设备的环境与运行数据
- 模型训练：批量训练、迁移学习、多作物建模
- 智能体分析：自动提炼各类植物数学模型与最佳参数策略
- 反馈闭环：训练结果回传边缘设备，用于下一轮调参更新

### 4. 云端平台模块

#### 4.1 数据管理与备份

- 原始数据上传：TF卡备份、云端存储、数据检索
- 设备日志：运行记录、告警历史、调参策略结果
- 安全性：身份验证、加密传输、用户权限管理

#### 4.2 模型服务

- 模型版本管理：不同植物、不同品种、不同环境的模型
- 在线推送：支持将训练结果推送至边缘网关
- 评估与监控：模型精度、效果对比、训练成本分析

#### 4.3 商业交易与App商城

- 农产品上架：自动发布多余蔬菜、螺旋藻、种植套餐
- 订单管理：订单接收、发货提醒、库存同步
- 涂鸦商城：借助涂鸦内置APP商城功能实现产品销售入口

### 5. 用户交互模块

#### 5.1 远程APP

- 设备列表与状态监控
- 作物类型选择与自动模式设置
- 运行工况实时显示
- 手动控制与远程命令
- 交易入口与商城浏览

#### 5.2 本地控制界面

- 触控屏或按键菜单
- 设备启动/停止、模式切换、手动执行
- 本地环境与状态展示
- 紧急停止与故障提示

### 6. 硬件与打印模块

#### 6.1 传感器与执行器规范

- 标准化接口：I2C、UART、GPIO、模拟量
- 传感器布局与噪声隔离
- 安全与防护：防水、防尘、防电磁干扰

#### 6.2 3D打印结构与装配

- 机箱、导管、托盘、支架等3D打印部件
- 模块化设计：快速更换、扩展升级
- 印刷材料：PLA、PETG、ABS 等推荐

## 开源发布建议

1. 在 GitHub 上创建公共仓库
2. 使用 `README.md` 和 `docs/` 做项目入口
3. 按模块添加子目录和说明文件
4. 提供硬件图纸、固件样例、模型说明、APP原型
5. 明确贡献指南与开源协议
