# 神农平台架构文档

## 项目概述

神农平台是基于城市种植的分布式AI农业生产系统，采用"网关 + 独立设备 + 养殖设备"架构。

## 系统拓扑

```
涂鸦 T5 网关
├── 涂鸦 ZS3L 边缘推理
│   └── 养殖设备（STM32）
│       ├── 块茎种植机（土豆等）
│       ├── 藻类种植机（螺旋藻等）
│       ├── 叶菜种植机（蔬菜等）
│       └── 菌类种植机（蘑菇等）
└── 云端服务（涂鸦平台）

K230 播种机（独立设备）
└── 为所有养殖机提供播种服务
```

## 设备分类

### 网关
- 硬件：涂鸦T5 + ZS3L
- 框架：TUYACLAW
- 职责：网关控制、边缘推理、云端通信

### 独立设备
- 硬件：K230
- 框架：K230 SDK
- 职责：图像识别、播种控制
- 特点：独立运行，不接入网关

### 养殖设备（按食用部位分类）
- 硬件：STM32 + ZS3L
- 框架：PlatformIO
- 职责：执行养殖任务、数据上报、接收网关控制

## 数据流

### 养殖设备
1. STM32采集传感器数据
2. ZS3L进行边缘处理
3. 执行器执行控制
4. 关键数据上报网关

### 独立设备
1. K230独立运行图像识别
2. 识别结果本地输出
3. 可通过UART与养殖设备通信

## 开发流程

1. 完成共享库开发
2. 实现养殖设备固件
3. 实现独立设备控制逻辑
4. 接入网关系统
5. 集成测试
