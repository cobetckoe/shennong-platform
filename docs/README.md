## 🎯 核心规范总结

### 1. 项目结构规范

```
firmware/
├── edge/          ← T5 核心网关（duckyclaw 在此）
├── meshtastic/    ← Mesh 网络模块
├── storage/       ← TF 卡存储管理
├── control/       ← 本地控制逻辑
└── ota/          ← 固件升级机制
```

**关键点**：
- 模块化设计，功能清晰划分
- duckyclaw 为本地智能引擎核心
- 配置文件统一存放在 `config/` 目录

### 2. duckyclaw 规范

**核心职责**：
1. 实时传感器数据采集与过滤
2. 本地策略规则评估与执行
3. 执行器控制指令下发
4. 数据存储与日志记录

**关键文件**：
- `duckyclaw_config.json` - 运行时配置
- `policy_rules.json` - 控制策略定义
- `src/core/policy_engine.c` - 策略引擎

### 3. Meshtastic 集成规范

**消息类型**（6 种）：
- `MSG_TYPE_SENSOR_DATA` - 传感器数据
- `MSG_TYPE_CONTROL_CMD` - 控制指令
- `MSG_TYPE_STATUS_HEARTBEAT` - 心跳
- `MSG_TYPE_ALERT` - 告警消息
- `MSG_TYPE_OTA_TRIGGER` - 升级触发
- `MSG_TYPE_CONFIG_UPDATE` - 配置更新

**射频参数**（中国）：
- 频率：470MHz
- 扩频因子：9
- 带宽：62.5kHz

### 4. 存储与 OTA 规范

**TF 卡结构**：
```
/sdcard/
├── data/       ← 传感器数据与事件日志
├── models/     ← TFLite 模型文件
├── config/     ← 配置备份
├── logs/       ← 运行日志
└── firmware/   ← OTA 升级包
```

**OTA 流程**：检查 → 下载 → 验证签名 → 备份 → 刷写 → 重启 → 健康检查

### 5. TuyaOS 约定

**物模型**：定义所有设备属性、事件、服务
**编译**：使用 CMake + 涂鸦 SDK
**MQTT**：遵循 tylink:// 主题规范
**版本**：使用 Semantic Versioning (MAJOR.MINOR.PATCH)

## 📞 技术支持

### 常见问题速查

**问题**：配置文件不生效
- **解决**：检查 JSON 语法，使用在线 JSON 验证工具

**问题**：duckyclaw 策略不执行
- **解决**：验证 policy_rules.json 中的 enabled 标志和条件配置

**问题**：Meshtastic 无法通信
- **解决**：检查串口配置、波特率、PSK 密钥是否一致

**问题**：TF 卡写入失败
- **解决**：检查磁盘空间和文件权限，运行 fsck 修复

## 📌 重要提醒

### ⚠️ 必读事项

1. **安全性**：
   - OTA 固件必须签名验证
   - Meshtastic PSK 密钥必须保密
   - 云端通信必须使用 TLS 加密

2. **兼容性**：
   - 所有 JSON 配置文件遵循 JSON Schema
   - 固件版本遵循 Semantic Versioning
   - 外部 API 采用 RESTful 设计

3. **可维护性**：
   - 代码注释使用 Doxygen 格式
   - 所有配置项都应有默认值
