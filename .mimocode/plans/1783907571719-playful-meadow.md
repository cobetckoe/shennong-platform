# 神农系统 T5 网关重构计划

## 目标

将 WBR3+ZS3L 网关替换为 T5 一体化网关，新增本地自动化引擎，支持断网离线控制。

## 架构变更

### 新数据流

```
用户APP → 涂鸦云 ←→ T5网关(虚拟子设备) ←→ Zigbee ←→ 子设备
                    ↕ (本地规则引擎)
             T5网关本地决策 → Zigbee → 子设备

涂鸦云中的其他设备(智能开关/传感器等) ↔ 涂鸦智能场景 ↔ T5网关虚拟子设备
```

### 虚拟子设备（跨设备联动关键）

T5 网关通过涂鸦 `tuya_iot_subdev` API 将每个 Zigbee 子设备注册为独立的涂鸦虚拟设备：

```
涂鸦 APP 看到的设备列表：
├── 神农系统网关 T5 (主设备)
│   ├── 根菜种植机 #1 (虚拟子设备 - 温湿度/水泵/风扇)
│   ├── 叶菜种植机 #2 (虚拟子设备 - 温湿度/喷雾/补光)
│   └── ...
├── 某品牌智能插座 (独立涂鸦设备)
├── 某品牌温湿度传感器 (独立涂鸦设备)
└── 某品牌智能开关 (独立涂鸦设备)
```

这样涂鸦 APP 的"智能场景"可以跨产品联动：
- 智能开关打开 → 触发神农系统网关开补光灯
- 独立温湿度传感器 > 30°C → 触发神农系统网关开风扇
- 神农系统网关检测到湿度低 → 触发智能加湿器开启

### 手机 APP 控制

APP 可以通过两种方式控制子设备：
1. **云端链路**：APP → 涂鸦云 → T5 → Zigbee → 子设备（在线时）
2. **局域网链路**：APP → WiFi → T5 → Zigbee → 子设备（同局域网时）
3. **离线自动化**：T5 本地执行规则，无需 APP 参与

## 开发环境

- **网关**：Tuya Wind IDE（涂鸦官方 IDE，图形化配置 T5 开发板）
- **子设备**：PlatformIO + STM32Cube 框架

## 本地自动化引擎设计（定时+阈值+联动）

### 自动化场景结构
```c
typedef struct {
    uint8_t scene_id;           // 场景ID (1-32)
    char name[32];              // 场景名称
    bool enabled;               // 是否启用

    // === 触发条件 (三选一) ===
    uint8_t trigger_type;       // 0=定时, 1=传感器阈值, 2=联动

    // 定时触发
    struct {
        uint8_t hour;           // 小时 (0-23)
        uint8_t minute;         // 分钟 (0-59)
        uint8_t repeat;         // 重复: bit0-6=周一到周日, 0xFF=每天
    } timer;

    // 传感器阈值触发
    struct {
        uint8_t device_id;      // 源设备 (0xFF=任意设备)
        uint8_t dpid;           // 数据点ID
        int32_t threshold_min;  // 下限 (含)
        int32_t threshold_max;  // 上限 (含)
    } sensor;

    // 联动触发 (设备A状态变化 → 触发设备B)
    struct {
        uint8_t source_device;  // 源设备ID
        uint8_t source_relay;   // 源设备继电器位
        uint8_t source_state;   // 0=关闭时触发, 1=开启时触发
    } link;

    // === 执行动作 ===
    uint8_t target_device;      // 目标设备 (0xFF=所有设备)
    uint8_t action_type;        // 0=设置继电器, 1=下发规则
    uint8_t action_relay_mask;  // 继电器掩码 (action_type=0时)
    uint8_t action_rule_id;     // 规则ID (action_type=1时)

    // === 冷却 ===
    uint8_t cooldown_sec;       // 触发后冷却时间 (秒)
    uint32_t last_triggered;    // 上次触发时间戳
} gw_scene_t;
```

### gw_subdev_manager.c/h — 虚拟子设备管理（跨设备联动核心）

```c
// 子设备信息
typedef struct {
    uint8_t zigbee_id;              // Zigbee 子设备 ID
    uint8_t dev_type;               // 设备类型 (DEVICE_ROOTCROP=0x01 等)
    char subdev_name[32];           // 虚拟设备名称
    uint8_t subdev_id;              // 涂鸦子设备实例 ID (网关内部分配)
    bool registered;                // 是否已注册到涂鸦云
    sensor_data_t last_sensor;      // 最新传感器数据
    uint8_t last_relay;             // 最新继电器状态
} gw_subdev_t;

// === 虚拟子设备生命周期 ===
void gw_subdev_manager_init(void);
void gw_subdev_manager_loop(void);

// 子设备上线时调用 → 自动注册虚拟设备到涂鸦云
void gw_subdev_on_online(uint8_t zigbee_id, uint8_t dev_type, const char *name);

// 子设备离线时调用
void gw_subdev_on_offline(uint8_t zigbee_id);

// === 数据同步到涂鸦云 ===

// 传感器数据上报 → 同步到虚拟子设备的 DP 点
void gw_subdev_report_sensor(uint8_t zigbee_id, const sensor_data_t *data);

// 继电器状态上报 → 同步到虚拟子设备的 DP 点
void gw_subdev_report_relay(uint8_t zigbee_id, uint8_t relay_mask);

// === 跨设备联动支持 ===

// 获取指定子设备的最新传感器数据（供 rule_engine 使用）
const sensor_data_t* gw_subdev_get_sensor(uint8_t zigbee_id);

// 获取指定子设备的继电器状态（供 rule_engine 使用）
uint8_t gw_subdev_get_relay(uint8_t zigbee_id);

// 获取所有在线子设备数量
uint8_t gw_subdev_get_count(void);

// 获取子设备信息
const gw_subdev_t* gw_subdev_get(uint8_t index);
```

#### 虚拟子设备注册流程
```
1. 子设备 Zigbee 心跳到达网关
2. gw_subdev_on_online() 被调用
3. 查找或创建 gw_subdev_t 记录
4. 调用 tuya_iot_subdev_init() 注册虚拟子设备
5. 配置虚拟子设备的 DP 点（传感器 + 执行器）
6. 涂鸦云将此虚拟设备添加到 APP 设备列表
7. 后续传感器/继电器数据通过 tuya_iot_subdev_dp_report() 上报
```

#### 每个虚拟子设备暴露的 DP 点
| DP ID | 名称 | 类型 | 说明 |
|-------|------|------|------|
| 1 | 温度 | VALUE | 环境温度 (×10) |
| 2 | 湿度 | VALUE | 环境湿度 (×10) |
| 3 | 光照 | VALUE | 光照强度 (lux) |
| 4 | pH值 | VALUE | 土壤pH (×10) |
| 5 | 水泵 | BOOL | 开/关 |
| 6 | 风扇 | BOOL | 开/关 |
| 7 | 补光灯 | BOOL | 开/关 |

## 本地自动化引擎工作原理

### 触发源
1. **传感器阈值**：温度 > 30°C → 开风扇
2. **定时**：每天 6:00 开补光灯，18:00 关
3. **联动**：设备 A 湿度低 → 设备 A 和 B 同时开喷雾
4. **云端更新**：APP 修改场景后立即生效

### 执行流程
```
gw_rule_engine_loop() {
    1. 获取当前时间 (涂鸦系统时间)
    2. 遍历所有自动化场景 (最多32个)
    3. 跳过未启用的场景
    4. 根据 trigger_type 检查触发条件:
       - 定时: 匹配 hour/minute/repeat
       - 阈值: 传感器值在 [min, max] 范围内
       - 联动: 源设备继电器状态匹配
    5. 检查冷却时间
    6. 满足条件 → 执行动作:
       - 设置继电器: 直接下发 CMD_DP_UPDATE
       - 下发规则: 下发 CMD_RULE_UPDATE
    7. 更新 last_triggered 时间戳
}
```

### 断网行为
- T5 持续运行本地规则引擎
- 子设备继续执行本地控制逻辑（device.c 中的 control_engine）
- T5 只是额外提供网关级自动化（跨设备联动）
- 网络恢复后同步状态到云端

## 验证方法

1. **编译验证**：所有工程可独立编译通过
2. **网关功能验证**：
   - T5 启动后可被涂鸦 APP 发现
   - 子设备上线后自动注册为虚拟子设备，出现在 APP 设备列表
   - 云端下发规则到网关，网关转发到子设备
   - 断网后本地规则引擎继续工作
3. **虚拟子设备验证**：
   - 每个子设备在 APP 中显示为独立设备（有温度/湿度/水泵/风扇等 DP 点）
   - APP 可单独查看每个子设备的传感器数据
   - APP 可单独控制每个子设备的执行器
4. **跨设备联动验证**：
   - 在 APP 创建智能场景：当独立温湿度传感器 > 30°C 时，触发神农系统网关开风扇
   - 在 APP 创建智能场景：当神农系统网关检测到湿度低时，触发智能加湿器开启
   - 在 APP 创建智能场景：智能开关打开时，触发神农系统网关开补光灯
5. **APP 验证**：
   - 在线时 APP 可查看所有子设备传感器数据
   - APP 可手动控制子设备
   - APP 可创建/修改自动化场景（本地 + 云端）
